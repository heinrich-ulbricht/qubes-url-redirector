Open in Qube

Welcome!

Hi, and thanks for installing Open in Qube! This page will help you to use this extension effectively.
"Open in Qube" is a browser extension that block and eventually redirect to another qube, URLs that doesn't match against a whitelist entry.

Usage: Toolbar button

The browser toolbar button has two functions:
  • It warns you, with a red icon, if the extension is not ready or some fatal error occurs. In the latter case reload the extension.
  • It allows you to access the settings page with a click.

Usage: Settings

Through the settings page you can customize how the extension handle non whitelisted URLs. By default it simply blocks the request. However it could redirect to a disposable qube or to a default one. Furthermore it could be disabled completely.

Usage: Whitelist

The whitelist is the most critical part of the extension. It allows you to specify if an URL will be opened in the current qube. Furthermore you can assign a descriptive label and a trust level to the whitelist entries.
Initially the whitelist is empty. So every URL will be blocked. For this reason before you start to surf the web you should start to populate the whitelist.
The whitelist is accessible through the settings page and its entries are of three types.
The three types of entries are:
  • RegExp
  • Exact Match
  • URL
Every entry type affects how an URL will be matched.

The "RegExp" type

This is the most flexible type. In fact you can describe, through JavaScript RegExp, whitelisted URL(s). Special careful should be used while defining an entry of this type. The "Exact Match" type

This is the most restrictive type. In fact with this type you can specify a string that will be matched against an URL if and only if they are equally. Here a screenshot of the input fields:

The "URL" type

This type allows you to add an entry specifying URL's componenent. Specifically:
  • Scheme, http or https. It's optional.
  • Host, it supports: IPv4, IPv6 and DNS name
  • Port. It's optional
  • Pathname, query string and fragment. It's optional
